# FlōGolf System Prototype Plan

Prepared March 12, 2026.

## Goal

Build a small interactive `FlōGolf System` demo that makes the owner feel the business is already operating inside one internal platform. The prototype should sell consolidation, control, and multi-location scale, not generic automation.

## What To Build

A single-screen interactive prototype with 5 core views:

1. Owner Dashboard
2. Customer 360
3. Event Pipeline
4. Automation Center
5. Multi-Location View

## Narrative

The prototype should tell this story:

- FlōGolf already has demand.
- Revenue is leaking between fragmented tools and manual handoffs.
- The `FlōGolf System` fixes that by owning the full customer lifecycle.
- Once that system exists, adding locations becomes much easier.

## Core Views

### 1. Owner Dashboard

Show:

- today's bookings
- event leads
- expiring memberships
- lesson follow-ups
- revenue at risk

### 2. Customer 360

Show one customer profile combining:

- bay bookings
- lessons
- memberships
- league history
- event inquiries
- spend
- automation status

### 3. Event Pipeline

Show how an event lead moves from:

- inquiry
- quote
- deposit
- booked

Include automated follow-up in the flow.

### 4. Automation Center

Show live workflows like:

- membership activation
- lesson scheduling
- no-show prevention
- renewal nudges
- win-back

### 5. Multi-Location View

Show Saugus plus a hypothetical second location to prove the system is built to scale.

## Key Demo Moments

These are the moments that should get him excited:

- Click a customer and see bookings, lessons, membership, and event history in one place.
- Click an event lead and show that follow-up, reminders, and deal stage are already handled.
- Show a membership purchase instantly activating access and triggering the next workflow.
- Flip from `Saugus` to `Location 2` and show the same system works there too.

## Demo Data

Use believable demo data:

- 4 to 6 customer profiles
- 3 to 4 event leads
- 2 active league players
- 2 expiring memberships
- 1 lesson package mid-usage
- 2 locations: `Saugus` and `Burlington` or `Woburn`

## Visual Direction

Make it feel premium and specific to indoor golf hospitality:

- dark green
- brass
- cream
- charcoal
- strong serif or display typography paired with a cleaner UI font
- subtle motion
- scoreboard-style metrics
- polished cards

Avoid generic blue SaaS dashboard styling.

## What Not To Build

Do not build:

- auth
- real backend
- real integrations
- admin complexity
- full CRM behavior

The prototype only needs enough interaction to make the system feel real.

## Build Sequence

1. Define the 5 screens and the story each one tells.
2. Write the exact on-screen copy before designing.
3. Sketch low-fi wireframes for all 5 screens.
4. Design one strong visual system.
5. Build the prototype with fake but realistic data.
6. Add 3 to 4 high-impact interactions.
7. Record a short walkthrough after it looks polished.

## Success Criteria

The prototype is working if he immediately understands:

- This would replace fragmented workflows.
- This would help us stop losing customers.
- This gives us control.
- This would make a second location easier.

## Talk Track

Use this while showing it:

> This isn’t a deck about software. This is what FlōGolf looks like when bookings, memberships, lessons, leagues, events, and retention all run through one system you own.
